Tabbed Editor Shell

The shared layout wrapper for post/page detail views

Route Structure

/admin/posts/$postId/          # Layout route (tab bar + header)
  ├── /edit                  # Content tab (existing editor)
  ├── /seo                   # SEO analysis dashboard
  ├── /traffic               # Traffic analytics
  └── /engagement            # On-page engagement

/admin/pages/$pageId/          # Same structure for pages
  ├── /edit
  ├── /seo
  ├── /traffic
  └── /engagement

Key Design Decisions

Tab bar shows live badges — SEO score (color-coded), total views, avg engagement time. Quick glanceable health without switching tabs.
Layout route loads post data once — The parent route fetches the post record and passes it to all child tabs via outlet context. No redundant queries.
Content tab is the existing editor — EditorLayout moves into the /edit child route unchanged. Zero disruption to what's already working.
Same structure for posts and pages — Identical tab layout, shared components. The content type is just a prop.
