# Type-honesty allowlist
#
# Suppressions in files matching these globs are accepted as
# documented framework limitations or deferred-domain debt, not
# unknown leakage. Patterns use glob syntax (** = any depth).
#
# Why the bulk of backend entries exist:
#   The Convex generated API + schema-derived union types hit
#   TypeScript's instantiation depth limit (TS2589) in strict mode
#   once the API surface grows past a moderate size. The accepted
#   workaround for this framework limitation is per-file
#   `@ts-nocheck` or per-line `@ts-expect-error`; the comments at
#   each suppression site cite TS2589 explicitly. This is tracked
#   in PRODUCTION-HUMAN-INTERVENTION.md as a Convex framework
#   limitation rather than per-domain technical debt.
#
# Why the test entries exist:
#   The Convex backend tsconfig deliberately excludes test files,
#   so Bun test globals (`test`, `expect`, etc.) trip the strict
#   editor checker. Tests still run via the Bun test runner.
