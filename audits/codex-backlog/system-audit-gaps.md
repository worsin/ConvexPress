6. **No EU VAT / VAT MOSS / OSS** — no digital-goods buyer-location logic, no VAT ID validation (VIES), no OSS reporting.
7. **No per-line tax storage on orders** — audit trail is a single scalar; compliance audits require per-rate breakdown.
8. **`taxRateBasis` setting is inert** — billing/shipping/store basis never read by calculator.
9. **`@ts-nocheck` on tax.ts** — masks type drift between rules table and calculator signature.
10. **Seed UI/backend mismatch** — alert mentions CA/GB/DE/AU rules that don't exist.
11. **No reporting** — no sales-by-jurisdiction export, no Form-1099-K or VAT return generation.
12. **No rate history / effective dates** — edits mutate live rules; no audit log of rate changes for retroactive order analysis.

Flag: **the PRD should be rewritten to match the KB scope**, or the commerce endpoints should also be surfaced under `/api/v1/` for headless commerce use cases. Currently headless commerce would have no public API.

6. **No per-request API log (`api_request_log`).** PRD table §4.1 defined per-request analytics (method, path, status, timing, IP, UA). Only webhook deliveries are logged at that granularity today. Per-key aggregates (`requestCount`, `lastUsedAt`, `lastUsedIp`) exist; per-call forensic logs do not.

7. **No admin analytics dashboard for API usage.** PRD §10 mentions a "View Usage" link per key. Data is available (`requestCount`, `lastUsedAt`, etc.), but no `/admin/analytics/api` route.

9. **Delivery action not re-invoked by Event Dispatcher on retry** — needs verification. The KB claims 3 retries w/ exponential backoff via Event Dispatcher; `createWebhook` sets the listener accordingly, but there is no explicit test here confirming the dispatcher actually schedules retries on non-2xx deliveries. Worth a smoke test.

10. **`requireCan` capabilities `api.*` must be seeded.** Mutations gate on `api.create_key`, `api.revoke_key`, `api.create_webhook`, `api.update_webhook`, `api.delete_webhook`. These must exist in the Role & Capability System's seed — quick cross-check with the roles system recommended.
